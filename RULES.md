# 项目开发规范 (Development Rules)

为了保持本项目工具的轻量级、便携性和易维护性，所有新增工具的开发必须严格遵循以下规则：

## 1. 代码结构
*   **单文件原则**：所有代码（HTML、CSS、JavaScript）必须合并在单个 `.html` 文件中。
    *   **CSS**：应放置在 `<head>` 标签内的 `<style>` 块中。
    *   **JavaScript**：应放置在 `<body>` 标签底部的 `<script>` 块中。
*   **零构建**：工具应当是可以直接在浏览器双击打开运行的，无需构建步骤（如 Webpack、Vite 等），无需本地服务器（除非特定功能需要）。

## 2. 技术选型
*   **原生开发**：使用原生 JavaScript (Vanilla JS)、HTML5 和 CSS3。
*   **禁止框架**：严禁使用 React、Vue、Angular 等现代前端框架。
*   **外部库**：尽量避免引入外部库。如果必须使用，优先选择 CDN 引入，但需确保在离线环境下核心功能尽可能可用（或明确标注需要网络）。

## 3. 文档要求
*   **独立文档**：每个工具必须有一个同名的 Markdown 说明文档（例如 `tool-name.html` 对应 `tool-name.md`）。
    *   文档内容应包含：功能介绍、使用方法、原始需求/提示词。
*   **索引更新**：每次新增工具后，必须在根目录的 `README.md` 文件中的“工具列表”部分添加相应的链接和描述。
